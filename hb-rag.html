임베딩이란?
텍스트를 의미를 담은 숫자 배열(벡터)로 변환하는 과정.
의미가 비슷한 문장은 벡터 공간에서 가까운 위치에 놓입니다. 
"손해배상 책임"
  → [0.023, -0.187, 0.445, ... ] ← 1024개 숫자

"배상 의무 규정"
  → [0.031, -0.172, 0.412, ... ] ← 가까움 ✅

"계약 해지 조건"
  → [0.612,  0.330, -0.12, ... ] ← 멀음 ❌

SQLite FTS5
Full-Text Search version 5 — SQLite 내장 전문 검색 엔진
FTS5 란?
FTS5 = Full-Text Search version 5
SQLite에 내장된 전문 검색 확장 모듈.
별도 서버 없이 파일 하나로 BM25 키워드 검색 제공.
한 줄 요약
Elasticsearch의 경량 온프레미스 버전.
동일한 BM25 알고리즘, 제로 서버 비용.
BM25 점수 공식
BM25(q, d) = Σ IDF(qᵢ) · TF(qᵢ, d) · (k₁+1)
─────────────────────────────
TF + k₁(1 - b + b·|d|/avgdl)
TF: 단어 빈도 | IDF: 역문서 빈도
|d|: 문서 길이 | avgdl: 평균 문서 길이
k₁=1.2, b=0.75 (FTS5 기본값)
일반 SQL vs FTS5 비교
-- ❌ 일반 LIKE 검색 → 느리고 전체 스캔 필요
SELECT * FROM docs
WHERE content LIKE '%손해배상%';

-- ✅ FTS5 → 역색인 기반, BM25 점수까지 반환
SELECT content, bm25(docs_fts) AS score
FROM  docs_fts
WHERE docs_fts MATCH '손해배상'
ORDER BY score;

-- AND / OR / NEAR 복합 검색도 지원
WHERE docs_fts MATCH '위약금 AND 30일'
WHERE docs_fts MATCH 'NEAR(손해 배상, 5)'

RRF 점수 공식
RRF_score(d) =
  α · Dense_score(d)
 + β · FTS5_score(d)
Dense Score × α + FTS5 Score × β = Hybrid Score
α, β 가중치는 도메인에 따라 조정.
계약서: α=0.7, β=0.3 (의미 우선)
로그 분석: α=0.4, β=0.6 (키워드 우선)

FTS5 테이블 생성 & 검색
-- FTS5 가상 테이블 생성
CREATE VIRTUAL TABLE contract_fts
USING fts5(
  chunk_id,
  article,
  content,
  tokenize = 'unicode61'  -- 한국어 지원
);

-- 청크 삽입
INSERT INTO contract_fts
VALUES('art5_sec1', '제5조', '계약 당사자가...');

-- 검색 + BM25 점수
SELECT chunk_id, content,
       bm25(contract_fts) AS score
FROM   contract_fts
WHERE  contract_fts MATCH '위약금 AND 30일'
ORDER BY score LIMIT 20;
쿼리 유형별 검색 경로
💬 "위약금 조항 알려줘"
→ Dense 강함 (의미 파악) + Sparse 보조
→ α=0.7, β=0.3 가중 적용
💬 "제5조 위약금 30일 이내"
→ Sparse 강함 (정확 키워드/숫자) + Dense 보조
→ α=0.3, β=0.7 가중 적용
💬 "손해배상 시 배상 한도는?"
→ Dense: "배상 한도" 의미 벡터 검색
→ Sparse: "손해배상", "한도" 키워드 매칭
→ RRF 통합 → Reranker → LLM 답변
